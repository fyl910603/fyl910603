Class可以通过extends关键字实现继承 

<span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">Point</span> {</span>
    <span class="hljs-comment">/*...*/</span>
}
<span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">ColorPoint</span> <span class="hljs-keyword"><span class="hljs-keyword">extends</span></span> <span class="hljs-title">Point</span>{</span>
    constructor(x,y,color){
        <span class="hljs-keyword">super</span>(x,y);<span class="hljs-comment">//调用父类的constructor(x,y)</span>
        <span class="hljs-keyword">this</span>.color = color;
    }
    toString(){
        <span class="hljs-keyword">return</span> <span class="hljs-keyword">this</span>.color + ' ' + <span class="hljs-keyword">super</span>.toString();<span class="hljs-comment">//调用父类的toString方法</span>
    }
}
`</pre>

注意：
a.super关键字在这里表示父类的构造函数，用来新建父类的this对象
b.子类必须在constructor方法中调用super方法，否则新建实例时会报错，这是因为子类没有自己的this对象，而是继承父类的this对象，然后对其进行加工。不调用super方法，子类就得不到this对象

<pre>`class Point{/*<span class="hljs-keyword">...</span>*/}
class ColorPoint extends Point{
        constructor(){
        /*<span class="hljs-keyword">...</span>*/
        }
    }
let cp = new ColorPoint(); //ReferenceError
`</pre>

上面的代码中ColorPoint继承了父类Point，但是它的构造函数没有调用super方法，导致新建实例时报错
ES6的继承机制:先创造父类的实例对象this(所以必须先调用super方法),然后再用子类的构造函数修改this
如果子类没有定义constructor方法，这个方法会被默认添加

<pre>`    <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">ColorPoint</span> <span class="hljs-keyword"><span class="hljs-keyword">extends</span></span> <span class="hljs-title">Point</span>{</span>}
    <span class="hljs-comment">//等同于</span>
    <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">ColorPoint</span> <span class="hljs-keyword"><span class="hljs-keyword">extends</span></span> <span class="hljs-title">Point</span>{</span>
        constructor(...args){
            <span class="hljs-keyword">super</span>(...args);
        }
    }
`</pre>

注:a.不管有没显示定义，任何一个子类都有constructor方法
b.在子类的constructor中只有先调用super之后才可以使用this关键字,否则会报错。这是因为子类实例的创建，是基于对父类实例加工

### Object.getPrototypeOf()方法

用于从子类上获取父类.

<pre>`  <span class="hljs-built_in">Object</span>.getPrototypeOf(ColorPoint) === Point
        <span class="hljs-comment">//true</span>
`</pre>

### super

super可以当做函数使用，也可以当做对象使用，两种情况用法不同
1.用作函数调用时:代表父类的构造函数。ES6中要求子类的构造函数必须执行一次super函数

<pre>`    <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">A</span>{</span>}
    <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">B</span> <span class="hljs-keyword"><span class="hljs-keyword">extends</span></span> <span class="hljs-title">A</span>{</span>
        constructor(){
            <span class="hljs-keyword">super</span>();
        }
    }
 `</pre>

2.注:super虽然代表父类A的constructor,但是返回的是子类B的实例，即super内部的this指的是B，super()在这里相当于：A.prototype.constructor.call(this);
